Transparency — Catalogue update commitment

App: Swiss Citizenship Test · Developer: Vincenzo Casini · Last updated: 10 May 2026

Why this page exists

This app sources its question pools from official cantonal websites for 7 Swiss cantons. Because each canton publishes its catalogue differently and updates it on its own schedule, users and institutions need a way to verify how this app keeps its data in sync with official sources. This page documents the verification process, the declared service level, the reporting channel for discrepancies, and the audit trail available on request.

This commitment is voluntary. The app is not affiliated with any government authority. It is independently developed by a naturalization applicant.

Service Level — what we promise

Source verification: all 7 official cantonal sources are checked once every 24 hours via an automated continuous-integration pipeline. Checks run on a staggered schedule (between 03:00 and 07:00 UTC) to avoid concurrent writes to the audit log.

Drift detection latency: a change in any official source is detected within 24 hours of publication.

Standard refresh window: when a change is detected, the updated catalogue is published in a new app version within 7 days.

Emergency refresh window: critical issues (factual errors flagged by users or canton authorities) are addressed within 72 hours.

Metadata review: the legal basis and procedure scope for each canton is manually re-verified every 6 months or on-demand if the canton announces a procedural change.

How verification works

The pipeline is built in four layers:

  1. Scraping — automated parsers read the official PDF or API for each canton, normalize the question and answer content, and produce a structured snapshot.
  2. Refresh orchestration — when a refresh is triggered, the parser re-runs from scratch. Existing question IDs are preserved; new questions get fresh IDs; questions removed by the canton are marked deprecated (never silently deleted).
  3. Drift detection — a nightly job computes a content hash of every official source and compares it against the previous baseline. If the hash differs, the job flags a drift event.
  4. Audit log — every verification run, drift or not, is recorded in an append-only structured log (one JSONL file per canton). Each entry includes timestamp, source URL, response status, content hash, previous hash, and any detected changes.

Per-question provenance

Every question in the app carries structured metadata that records:

  • Extraction method (PDF text, OCR, manual curation, or API)
  • Extraction date and the hash of the source document at extraction time
  • Original answer format (true-false, MCQ, open) and the format actually delivered in the app
  • An adapted flag set when the delivered format differs from the original (for example, open questions reformatted as MCQ with alternative choices)

This metadata makes every question independently verifiable against its source.

Reporting discrepancies

If you find a question that is outdated, incorrect, or missing relative to its official source, please report it. Reports are processed within the SLA above — 72 hours for critical issues, 7 days for standard updates.

Audit log access

Every verification run is recorded in append-only structured logs (one JSONL file per canton). Logs include timestamp, source URL, response status, content hash, and any detected changes. Logs are kept indefinitely on a dedicated git branch.

The audit log is not published openly because it lives in the application's production repository. It is available for inspection by cantonal naturalization authorities, the State Secretariat for Migration, accredited researchers, or journalists on written request at swisscitizenshiptest@gmail.com. Inspection requests are honoured within 5 working days.

Versioning policy

Each catalogue snapshot is identified by:

  • An ISO date (when the snapshot was captured from the official source)
  • A content hash (computed over the normalized catalogue)
  • An app version (the first published app release that ships this snapshot)

Historical snapshots are preserved so any past version can be inspected and compared against the live one.

About this app

Swiss Citizenship Test is a free, ad-free, account-free study tool for the Swiss naturalization knowledge test. It is independently developed by a naturalization applicant living in Switzerland. It is not affiliated with any government authority. Its purpose is to complement official preparation tools — not to replace them.
